IMBannerDelegate Protocol Reference

Conforms to NSObject
Declared in IMBannerDelegate.h

Overview

This is the delegate for receiving state change messages from an IMBanner. Use this to receive callbacks for banner ad request succeeding, failing or for the events after the banner ad is clicked.

banner:didFailToReceiveAdWithError:

Callback sent when an ad request failed. Normally this is because no network connection was available or no ads were available (i.e. no fill).

- (void)banner:(IMBanner *)banner didFailToReceiveAdWithError:(IMError *)error

Parameters

banner

The IMBanner instance that failed to load the ad request.

The error that occurred during loading.

bannerWillLeaveApplication:

Callback sent just before the application goes into the background because the user clicked on a link in the ad that will launch another application (such as the App Store). The normal UIApplicationDelegate methods like applicationDidEnterBackground: will immediately be called after this.

- (void)bannerWillLeaveApplication:(IMBanner *)banner

Parameters

banner

The IMBanner instance that is launching another application.

bannerWillPresentScreen:

Callback sent just before when the banner is presenting a full screen view to the user. Use this opportunity to stop animations and save the state of your application in case the user leaves while the full screen view is on screen (e.g. to visit the App Store from a link on the full screen view).

- (void)bannerWillPresentScreen:(IMBanner *)banner

Parameters

banner

The IMBanner instance that presents the screen.
